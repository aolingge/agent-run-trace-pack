## Codebase Patterns
- Package preview contract is defined by `package.json` `files` plus `scripts/release-dry-run.mjs`, which uses `npm pack --dry-run --json`; keep `docs/release-readiness.md` in sync with both.
- Synthetic trace fixtures live under `tests/fixtures/synthetic-traces/` as data-only JSON files; load them in sorted filename order and keep fixture commands as input data, not executable test steps.
- CLI tests import `main` from `src/cli.ts`; keep the direct-run guard so importing the CLI does not execute process-exit behavior.
- Run detectors on raw command/output, then redact command fields and finding evidence before writing manifest or reports.
- Report rendering fixtures live in `tests/fixtures/report-manifest.json` plus `tests/fixtures/expected-reports/`; verify HTML through localhost when Playwright blocks direct `file:` URLs.
- Launch docs are split between `docs/examples.md` for synthetic source-mode commands and `docs/launch-kit.md` for maintainer review; keep README and `docs/index.html` linked to quick start, examples, safety, report preview, and release readiness.
